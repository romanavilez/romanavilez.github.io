Why a Child Safety App

When I started to think about my capstone, one thing was clear: I wanted to build an app that would make the world a safer place. It couldn't just be any app; it had to be different and something that society would actually want to use. The problem was that there are already so many apps out there that focus on safety, so chances were that if I thought of something, it would already exist. As I was brainstorming, I remembered an app that I had worked on during a hackathon at UWB; it was an app that would reroute a user through a safer path if they were approaching an unsafe location. So, then I thought: what if I built something similar but with a different goal? Then an idea came to mind: an app that would keep children out of danger, with the same rerouting mechanic, but instead be a centralized place for parents

The idea was there, but there was still something that this app needed, something that would make this app stand out. I liked the idea of the end users being parents and children, but rerouting is something that is already out there, so I decided to ditch that idea. That's when I thought: what if I could create an app that gave children autonomy while still keeping them safe? An app that would give children the freedom to explore while still providing parents with the tools they need to keep their children safe. And that's how ChildWay was born.
